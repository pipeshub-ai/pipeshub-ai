# 03 — Three-page product brief (docx)

## User message

> Write a 3-page product brief as a .docx for a new feature called
> "Scheduled Exports". Sections: Problem, Proposed Solution, API Changes,
> Rollout Plan, Success Metrics. Include a risks section. Use an editorial
> theme with serif headings.

## What "great" looks like

- Cover page with title (large serif), subtitle, author, date, and a thin
  colored divider rule.
- Page numbering in the footer ("Page N of M") and a muted header with the
  document title.
- Heading 1 for each major section, Heading 2 for subsections.
- A callout box for the "Headline" line in the Problem section.
- A 3-column table in "API Changes" (Endpoint, Method, Purpose) with subtle
  row shading and primary-color header row.
- A bulleted Risks list using library numbering (not manual "•").

## Common failure modes to watch for

- Page size defaults to A4 when user expected US Letter (or vice versa).
- Table uses percentages instead of DXA widths and breaks in Google Docs.
- Heading hierarchy collapsed (everything bold, no outline).
- Missing footer or page numbers.
