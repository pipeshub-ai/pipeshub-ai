# 05 — Data-heavy analyst report (docx)

## User message

> Produce a 6-page analyst-style report on the company's user-growth trends
> for FY25. Monthly active users by quarter: Q1 120k, Q2 145k, Q3 180k,
> Q4 235k. Include a retention cohort table (6 cohorts × 4 months). Write an
> executive summary, methodology, findings, recommendations, and appendix.
> Use a teal/trust theme.

## What "great" looks like

- Cover page, TOC, executive summary callout box.
- MAU chart embedded (a PNG produced by matplotlib or chart.js saved as image
  then inserted) with caption.
- Retention table (6×5 with header row) using proper DXA widths, zebra
  rows, primary-color header.
- At least two heading levels.
- Proper footer page numbering.
- Methodology section uses a numbered list.
- Findings section uses bold inline headings (Heading 3) with paragraphs.

## Common failure modes to watch for

- No chart — just bullet points describing the numbers.
- Retention table missing or collapsed into text.
- Everything in one giant paragraph block with no heading structure.
